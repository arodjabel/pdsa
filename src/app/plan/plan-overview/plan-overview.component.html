Quality Overview


Set an aim for the Task Force

Should be set by the Quality Improvement Committee and SMART. Is it Quality Improvement or Quality Assurance? Consider: which systems will be affected by the improvement efforts? What can the Task Force do or not do without the QI Committee?

Form the Team

Include Clinical Leader, Technical Expert, a Day-to-Day Leader, a Project Leader, and additional members. Consider Board Members and/or patients, if applicable.

Not Started In Progress Complete
Establish Measures

What are we trying to accomplish? How will we know that a change is an improvement? What possible changes can we make that will result in improvement? Use SMART measures when possible.

Not Started In Progress Complete
Selecting Changes

What possible changes can we make to meet and exceed measures? Some possible categories of changes are: eliminate waste, improve workflow, optimize inventory, change the work environment, patient experience, improve time management, reduce variation, error proofing, or service improvement. Complete a SWOT Form.

Not Started In Progress Complete
Testing Changes (Pilot)

Use PDSA Form. Repeat until a process or change is defined and shows improvement.

Not Started In Progress Complete
Implementing Changes

Formalize the change. What needs to be updated before spreading to the entire organization? Examples: updated workflows, policies, procedures, training guides, eClinicalworks, job descriptions, consent forms, etc.

Not Started In Progress Complete
Spreading the Change

Expand the change to the entire organization. Does the project need to be moved to Quality Assurance to be monitored? How often will the measures be revisited? How will we know when the process needs to be put back into Quality Improvement?
